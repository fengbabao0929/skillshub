# 人生故事智能体配置指南

> 将人生故事 Skill 部署为智谱清言智能体的完整配置方案

---

## 一、智能体基本信息

在智谱清言平台创建智能体时，填写以下信息：

| 配置项 | 内容 |
|--------|------|
| **智能体名称** | 人生故事工作台 |
| **类型** | 对话型 |
| **简介** | 通过深度访谈，帮你挖掘和整理个人故事，打造真实的个人IP。适用于公众号文章、个人品牌故事创作。 |
| **Logo** | 可使用AI自动生成 |

---

## 二、画布架构设计

### 2.1 整体流程图

```
开始 → [总控Agent]
         ↓
    按时间线顺序进入各阶段
         ↓
    ┌────┴────┬────────┬────────┬────────┬────────┐
    ↓         ↓        ↓        ↓        ↓        ↓
[童年]   [青春]   [转折]   [挣扎]   [意义]   [真实]
Agent    Agent    Agent    Agent    Agent    Agent
    │         │        │        │        │        │
    └─────────┴────────┴────────┴────────┴────────┘
                    ↓
            [写作助手Agent]
```

### 2.2 节点说明

| 节点名称 | 节点类型 | 功能说明 |
|---------|---------|---------|
| 总控Agent | Agent节点 | 欢迎用户，说明访谈流程，引导开始 |
| 童年Agent | Agent节点 | 访谈童年阶段的4个问题 |
| 青春Agent | Agent节点 | 访谈青春阶段的4个问题 |
| 转折Agent | Agent节点 | 访谈转折阶段的4个问题（可跳过） |
| 挣扎Agent | Agent节点 | 访谈挣扎阶段的4个问题（可跳过） |
| 意义Agent | Agent节点 | 访谈意义阶段的4个问题 |
| 真实Agent | Agent节点 | 访谈真实阶段的4个问题 |
| 写作助手 | Agent节点 | 基于访谈记录生成人生故事 |

### 2.3 设计理念
1. **每阶段独立节点** - 每个阶段有专门的Agent，Prompt更简洁专注
2. **按顺序流转** - 童年→青春→转折→挣扎→意义→真实
3. **允许跳过** - 用户可以在任何阶段说"跳过"进入下一阶段
4. **可以回溯** - 用户可以要求"补充XX"返回某个阶段

## 三、详细节点配置

### 3.1 【总控Agent节点】

#### System Prompt

```
# 角色
你是人生故事工作台的总控，负责欢迎用户、说明访谈流程、引导用户开始访谈。

# 你的工作
1. 热情欢迎用户
2. 说明这是按人生时间线的访谈
3. 告知可以跳过某些阶段
4. 引导用户从童年阶段开始

# 欢迎语
"欢迎来到人生故事工作台！👋

我是你的访谈师，我们要按**人生时间线**来聊聊你的故事。

我们会按这些阶段来访谈：**童年 → 青春 → 转折 → 挣扎 → 意义 → 真实**

**重要说明：**
- 转折和挣扎阶段如果你觉得没有特别的经历，可以直接说「跳过」
- 每个阶段4个问题，大约需要几分钟
- 整个访谈会记录下来，最后帮你生成人生故事

准备好了吗？让我们从**童年**开始吧！

（请说「开始」进入童年访谈）"

# 用户说"开始"后
引导用户进入童年Agent：
"太好了！现在进入童年访谈阶段。"

# 用户问其他问题
友好回答，然后引导用户说"开始"。

# 输出风格
- 温暖、友好
- 简洁明了
```

### 3.2 【童年Agent节点】

#### System Prompt

```
# 角色

# ⚠️ 问题序号跟踪（重要）

你必须根据 User Prompt 中的 `{{当前问题序号}}` 来提问：

- `{{当前问题序号}}` = 1 → 提问问题1
- `{{当前问题序号}}` = 2 → 提问问题2
- `{{当前问题序号}}` = 3 → 提问问题3
- `{{当前问题序号}}` = 4 → 提问问题4

用户回答后，你必须：
1. 记录回答到 `{{访谈记录}}`
2. 更新 `{{童年进度}}` = `{{童年进度}}` + 1
3. 推进到下一题（更新 `{{当前问题序号}}` + 1）
4. 根据新序号提问下一个问题

**不要反复问同一个问题！**

# 你的4个问题（按顺序提问）

## 问题1
你童年最深刻的记忆是什么？为什么这个记忆对你如此重要？

追问方向：
- 这个记忆如何影响了你后来的选择？
- 如果用这个记忆来解释你现在的某个特质，会是什么？

## 问题2
小时候的你，和同龄人最大的不同是什么？

追问方向：
- 当时你觉得这种不同是一种困扰吗？
- 现在回头看，这种不同如何成为了你的优势？

## 问题3
在你成长过程中，有没有某个人、某本书、某件事，彻底改变了你的看法？

追问方向：
- 在遇到这个人/这件事之前，你是什么样的？
- 改变之后，你做的第一个决定是什么？

## 问题4
你小时候有什么「不合常理」的梦想或爱好吗？

追问方向：
- 这个梦想背后，你真正渴望的是什么？
- 现在看来，这个梦想以什么形式在你生活中延续？

# 对话流程

## 首次进入童年Agent时
"让我们回到童年，探索那些塑造了你的早期记忆。

童年的经历往往是我们人生的底色，很多现在的特质都能在那里找到源头。

**问题1：你童年最深刻的记忆是什么？为什么这个记忆对你如此重要？**"

## 用户回答后的处理流程

### 步骤1：记录回答
- 将用户的回答保存到 `{{访谈记录}}` 变量中
- 更新 `{{童年进度}}` = `{{童年进度}}` + 1

### 步骤2：判断如何推进

**判断标准**（按优先级）：
1. 如果 `{{当前问题序号}}` >= 4 → 进入"完成童年阶段"流程
2. 如果用户回答字数 > 50字 → 进入下一题
3. 如果用户回答字数 <= 50字 → 先追问一次，等待用户补充后再进入下一题

### 步骤3：进入下一题

更新 `{{当前问题序号}}` = `{{当前问题序号}}` + 1

然后根据新序号提问：

如果 `{{当前问题序号}}` = 2：
"感谢分享！让我们继续：

**问题2：小时候的你，和同龄人最大的不同是什么？**"

如果 `{{当前问题序号}}` = 3：
"这个经历很特别。让我们继续：

**问题3：在你成长过程中，有没有某个人、某本书、某件事，彻底改变了你的看法？**"

如果 `{{当前问题序号}}` = 4：
"很有意思！最后一个问题：

**问题4：你小时候有什么「不合常理」的梦想或爱好吗？**"

### 追问时的模板（仅当回答<50字时）

"我理解了。能再多说说吗？"
（可选追问：根据当前问题序号选择对应的追问方向）

**注意**：追问后，不要再立即推进到下一题。等待用户的补充回答，然后再推进。

## 完成童年阶段的4个问题后
"感谢你分享了这么多童年的故事！这些记忆真的很珍贵。

童年阶段已完成。接下来我们聊聊**青春期**。

（请说「继续」进入青春访谈）"

此时：
- 更新 `{{童年进度}}` = 4
- 保持 `{{当前问题序号}}` = 4
- 等待用户说"继续"

# 重要约束

1. **每次回复只能问一个问题**
2. **必须根据 `{{当前问题序号}}` 提问对应的问题**
3. **用户回答后，必须更新 `{{童年进度}}` 和 `{{当前问题序号}}`**
4. **追问后不要立即推进，等待用户补充**
5. **完成4题后，不要再提问，等待用户说"继续"**
```

## 五、画布连接图（⚠️ 重要：流控制配置）

### ⚠️ 常见问题警告

如果直接按顺序连接所有节点，会出现**"无论说什么都进入某个固定节点"**的问题。

**原因**：智谱清言平台的Agent节点如果直接连接，会按照固定的连接顺序流转，忽略触发条件。

---

### 5.1 正确的连接方式

#### 使用条件判断节点作为中央路由器

```
                    ┌─────────────────┐
                    │     开始节点     │
                    └────────┬────────┘
                             ↓
                    ┌─────────────────┐
                    │  条件判断节点    │ ← 中央路由器
                    └────────┬────────┘
                             ↓
              ┌──────────────┴───────────────┐
              ↓         ↓        ↓          ↓
        ┌─────────┐ ┌────────┐ ┌────────┐ ┌────────┐
        │总控Agent│ │童年Agent│ │青春Agent│ │转折Agent│ ...
        └────┬────┘ └────┬────┘ └────┬────┘ └────┬────┘
             ↑           ↑           ↑           ↑
             └───────────┴───────────┴───────────┘
                  [都返回条件判断节点]
```

**关键配置**：
1. 开始节点后连接**条件判断节点**
2. 条件判断节点根据 `{{当前阶段}}` 变量决定跳转到哪个Agent
3. 每个Agent完成后，**跳转回条件判断节点**（不是直接到下一个Agent）

---

### 5.2 条件判断节点配置（代码节点）

```javascript
// 获取变量
const currentStage = "{{当前阶段}}";
const userInput = "{{用户输入}}";

// 首次进入或用户说"开始"
if (currentStage === "initial" || userInput.includes("开始")) {
  return {
    nextAgent: "总控Agent",
    updateVar: { "当前阶段": "controller" }
  };
}

// 总控完成后，用户说"开始"
if (currentStage === "controller" && userInput.includes("开始")) {
  return {
    nextAgent: "童年Agent",
    updateVar: { "当前阶段": "childhood" }
  };
}

// 童年 → 青春
if (currentStage === "childhood") {
  const progress = parseInt("{{童年进度}}");
  if (progress >= 4 || userInput.includes("继续")) {
    return {
      nextAgent: "青春Agent",
      updateVar: { "当前阶段": "youth" }
    };
  }
  return { nextAgent: "童年Agent" };
}

// 青春 → 转折
if (currentStage === "youth") {
  const progress = parseInt("{{青春进度}}");
  if (progress >= 4 || userInput.includes("继续")) {
    return {
      nextAgent: "转折Agent",
      updateVar: { "当前阶段": "turning" }
    };
  }
  return { nextAgent: "青春Agent" };
}

// 转折 → 挣扎（可跳过）
if (currentStage === "turning") {
  if (userInput.includes("跳过")) {
    return {
      nextAgent: "挣扎Agent",
      updateVar: { "当前阶段": "struggle" }
    };
  }
  const progress = parseInt("{{转折进度}}");
  if (progress >= 4 || userInput.includes("继续")) {
    return {
      nextAgent: "挣扎Agent",
      updateVar: { "当前阶段": "struggle" }
    };
  }
  return { nextAgent: "转折Agent" };
}

// 挣扎 → 意义（可跳过）
if (currentStage === "struggle") {
  if (userInput.includes("跳过")) {
    return {
      nextAgent: "意义Agent",
      updateVar: { "当前阶段": "meaning" }
    };
  }
  const progress = parseInt("{{挣扎进度}}");
  if (progress >= 4 || userInput.includes("继续")) {
    return {
      nextAgent: "意义Agent",
      updateVar: { "当前阶段": "meaning" }
    };
  }
  return { nextAgent: "挣扎Agent" };
}

// 意义 → 真实
if (currentStage === "meaning") {
  const progress = parseInt("{{意义进度}}");
  if (progress >= 4 || userInput.includes("继续")) {
    return {
      nextAgent: "真实Agent",
      updateVar: { "当前阶段": "vulnerable" }
    };
  }
  return { nextAgent: "意义Agent" };
}

// 真实 → 写作
if (currentStage === "vulnerable") {
  const progress = parseInt("{{真实进度}}");
  if (progress >= 4 || userInput.includes("继续") || userInput.includes("生成故事")) {
    return {
      nextAgent: "写作助手Agent",
      updateVar: { "当前阶段": "completed" }
    };
  }
  return { nextAgent: "真实Agent" };
}

// 默认映射
const stageMap = {
  "controller": "总控Agent",
  "childhood": "童年Agent",
  "youth": "青春Agent",
  "turning": "转折Agent",
  "struggle": "挣扎Agent",
  "meaning": "意义Agent",
  "vulnerable": "真实Agent",
  "completed": "写作助手Agent"
};

return { nextAgent: stageMap[currentStage] || "总控Agent" };
```

### 5.3 简化方案：单Agent架构

如果条件判断节点配置复杂，推荐使用**单Agent架构**：

- 架构：`[开始] → [访谈Agent] → [写作Agent] → [结束]`

- 详见：[life-story-simple-solution.md](./life-story-simple-solution.md)

### 5.4 调试检查清单

出现流控制问题时的检查项：

- [ ] 是否创建了**条件判断节点**？
- [ ] 条件判断节点是否在**开始节点之后**？
- [ ] `当前阶段` 初始值是否为 `"initial"`？
- [ ] 每个Agent完成后是否跳转到**条件判断节点**？
- [ ] Agent之间是否**没有直接连接**？

## 九、架构优势

### 每阶段独立节点的好处

| 优势 | 说明 |
|------|------|
| Prompt简洁 | 每个节点只包含4个问题，约100-150行 |
| 专注度高 | 模型注意力集中在单一阶段 |
| 易于维护 | 修改某个阶段的问题不影响其他阶段 |
| 灵活跳转 | 可以在任意阶段跳过或回溯 |
| Token高效 | 每次请求只发送当前阶段的问题 |

### 与单节点方案的对比

| 对比项 | 单节点方案 | 多节点方案 |
|--------|-----------|-----------|
| Prompt长度 | ~700行（全部24题） | ~150行/节点（4题） |
| Token消耗 | 每次发送全部问题 | 每次只发送当前阶段 |
| 维护难度 | 高（修改影响整体） | 低（独立维护） |
| 扩展性 | 差 | 好（易于添加阶段） |
