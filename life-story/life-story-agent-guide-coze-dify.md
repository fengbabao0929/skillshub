# 快速开始：两种方案选择

在配置之前，请先选择适合你的方案：

## 方案A：单Agent架构（推荐新手）

**架构**：`开始 → [访谈Agent] → [写作Agent] → 结束`

**优势**：
- ✅ 配置简单，只需要2个Agent节点
- ✅ 不需要复杂的路由逻辑
- ✅ 调试容易，出错容易定位
- ✅ 状态管理集中

**适用场景**：
- 配置经验有限
- 希望快速部署
- 不需要复杂的功能扩展

**文档位置**：参见 [life-story-simple-solution.md](./life-story-simple-solution.md)

---

## 方案B：多Agent架构（推荐进阶用户）

**架构**：`开始 → [中央路由] → [总控] → [童年] → [青春] → [转折] → [挣扎] → [意义] → [真实] → [写作] → 结束`

**优势**：
- ✅ 每个阶段独立Agent，职责清晰
- ✅ 模块化程度高，易于维护
- ✅ 支持更复杂的扩展功能

**适用场景**：
- 有丰富的配置经验
- 需要更高的可维护性
- 计划添加复杂功能

**⚠️ 注意**：此方案需要正确配置中央路由器，否则会出现"无论说什么都进入青春节点"的问题。详见下方第一部分的配置步骤。

---

**建议**：如果你是第一次配置，或者遇到了流控制问题，请先使用**方案A（单Agent架构）**。

# 第一部分：Coze（扣子）平台配置

## 一、Coze 平台简介

Coze（扣子）是字节跳动推出的 AI Bot 开发平台，支持：
- 可视化工作流编排
- 多节点协作
- 插件扩展
- 变量管理
- 长期记忆

## 二、在 Coze 上创建智能体

### Step 1: 创建 Bot

1. 访问 https://www.coze.cn/
2. 点击"创建 Bot"
3. 选择"工作流 Bot"
4. 名称：`人生故事工作台`
5. 描述：`通过深度访谈，帮你挖掘和整理个人故事，打造真实的个人IP`

### Step 2: 配置基本信息

```
Bot 名称：人生故事工作台
Bot 简介：按人生时间线访谈，生成6000-8000字的个人IP故事
Bot 头像：可使用AI生成
```

| interview_data | array | [] | 访谈记录数组 |

| youth_count | number | 0 | 青春阶段计数 |
| turning_count | number | 0 | 转折阶段计数 |
| struggle_count | number | 0 | 挣扎阶段计数 |

### 3.4 节点连接逻辑（⚠️ 重要：正确配置）

#### ⚠️ 常见错误：直接连接所有节点

**错误的连接方式**（会导致"无论说什么都进入青春节点"）：
```
[开始] → [总控] → [童年] → [青春] → [转折] → ...
```

这种直接连接会导致系统默认总是流向第一个连接的节点（通常是青春节点）。

---

#### ✅ 正确的连接方式：使用中央路由器

**正确的架构**：
```
            [开始]
              ↓
         [中央路由节点]
              ↓
    ┌─────────┴─────────┐
    ↓    ↓    ↓    ↓    ↓    ↓    ↓
[总控] [童年] [青春] [转折] [挣扎] [意义] [真实] [写作]
    ↑    ↑    ↑    ↑    ↑    ↑    ↑
    └────────────────────┘
         [返回路由器]
```

---

#### 步骤1：创建中央路由器节点（条件分支节点）

**节点类型**：条件分支（Condition Branch）

**配置逻辑**：

```javascript
// 伪代码展示路由逻辑
if (当前阶段 == "初始" || user_input.contains("开始")) {
  return "总控节点";
}

if (当前阶段 == "总控" && user_input.contains("开始")) {
  更新变量(当前阶段 = "童年");
  return "童年节点";
}

if (当前阶段 == "童年") {
  if (childhood_count >= 4 || user_input.contains("继续")) {
    更新变量(当前阶段 = "青春");
    return "青春节点";
  } else {
    return "童年节点";
  }
}

if (当前阶段 == "青春") {
  if (youth_count >= 4 || user_input.contains("继续")) {
    更新变量(当前阶段 = "转折");
    return "转折节点";
  } else {
    return "青春节点";
  }
}

if (当前阶段 == "转折") {
  if (user_input.contains("跳过")) {
    更新变量(当前阶段 = "挣扎");
    return "挣扎节点";
  }
  if (turning_count >= 4 || user_input.contains("继续")) {
    更新变量(当前阶段 = "挣扎");
    return "挣扎节点";
  } else {
    return "转折节点";
  }
}

if (当前阶段 == "挣扎") {
  if (user_input.contains("跳过")) {
    更新变量(当前阶段 = "意义");
    return "意义节点";
  }
  if (struggle_count >= 4 || user_input.contains("继续")) {
    更新变量(当前阶段 = "意义");
    return "意义节点";
  } else {
    return "挣扎节点";
  }
}

if (当前阶段 == "意义") {
  if (meaning_count >= 4 || user_input.contains("继续")) {
    更新变量(当前阶段 = "真实");
    return "真实节点";
  } else {
    return "意义节点";
  }
}

if (当前阶段 == "真实") {
  if (vulnerable_count >= 4 || user_input.contains("继续")) {
    更新变量(当前阶段 = "完成");
    return "写作节点";
  } else {
    return "真实节点";
  }
}

if (当前阶段 == "完成" || user_input.contains("生成故事")) {
  return "写作节点";
}
```

#### 步骤2：Coze 条件分支节点的实际配置

在 Coze 中，使用"条件分支"节点，配置如下：

**分支1**：初始化检测
```
条件：{{current_stage}} == "initial" OR {{user_input}} contains "开始"
动作：跳转到 → [总控节点]
```

**分支2**：总控完成后
```
条件：{{current_stage}} == "controller" AND {{user_input}} contains "开始"
动作：
  1. 设置变量 {{current_stage}} = "childhood"
  2. 跳转到 → [童年节点]
```

**分支3**：童年阶段
```
条件：{{current_stage}} == "childhood"
子条件A：{{childhood_count}} >= 4 OR {{user_input}} contains "继续"
  动作：
    1. 设置变量 {{current_stage}} = "youth"
    2. 跳转到 → [青春节点]
子条件B：其他
  动作：跳转到 → [童年节点]
```

**分支4**：青春阶段
```
条件：{{current_stage}} == "youth"
子条件A：{{youth_count}} >= 4 OR {{user_input}} contains "继续"
  动作：
    1. 设置变量 {{current_stage}} = "turning"
    2. 跳转到 → [转折节点]
子条件B：其他
  动作：跳转到 → [青春节点]
```

**分支5**：转折阶段
```
条件：{{current_stage}} == "turning"
子条件A：{{user_input}} contains "跳过"
  动作：
    1. 设置变量 {{current_stage}} = "struggle"
    2. 跳转到 → [挣扎节点]
子条件B：{{turning_count}} >= 4 OR {{user_input}} contains "继续"
  动作：
    1. 设置变量 {{current_stage}} = "struggle"
    2. 跳转到 → [挣扎节点]

  动作：跳转到 → [转折节点]

**分支6**：挣扎阶段

子条件A：{{user_input}} contains "跳过"

    2. 跳转到 → [意义节点]

  动作：
    1. 设置变量 {{current_stage}} = "meaning"
    2. 跳转到 → [意义节点]
子条件C：其他
  动作：跳转到 → [挣扎节点]
```

**分支7**：意义阶段

条件：{{current_stage}} == "meaning"

    1. 设置变量 {{current_stage}} = "vulnerable"

  动作：跳转到 → [意义节点]

如果仍然出现"无论说什么都进入青春节点"的问题，请按以下清单检查：

- [ ] `current_stage` 变量的初始值是否设置为 `"initial"`？
- [ ] 是否创建了**中央路由节点**（条件分支节点）？
- [ ] 所有节点是否都连接到**中央路由节点**，而不是直接连接？
- [ ] 条件分支的顺序是否正确？（Coze 从上到下匹配第一个满足的条件）
- [ ] 是否在"跳过"判断之前检查了"继续"？（因为"跳过"的判断应该优先）
- [ ] 每个阶段节点是否正确更新了对应的计数变量？
- [ ] 条件判断中使用的变量名是否与定义的变量名完全一致？

---

### 3.6 测试流程

正确的测试流程应该是：

1. **首次输入任意内容** → 进入总控节点，看到欢迎语
2. **输入"开始"** → 进入童年节点，看到第1个问题
3. **回答问题** → 继续在童年节点，看到第2个问题
4. **输入"继续"** → 进入青春节点，看到青春第1个问题
5. **输入"跳过"**（在转折或挣扎阶段） → 跳过当前阶段，进入下一阶段

# 第三部分：平台差异对比

## 功能对比表

| 功能 | Coze（扣子） | Dify |
|------|-------------|------|
| **使用门槛** | 低，国内访问快 | 中，需魔法或私有部署 |
| **工作流编排** | 可视化画布 | 可视化画布 |
| **节点类型** | 丰富，支持插件 | 基础但够用 |
| **变量管理** | 内置变量系统 | 内置变量系统 |
| **长期记忆** | 支持 | 支持（需配置） |
| **知识库** | 支持 | 支持，向量检索强 |
| **API服务** | 支持 | 支持，更灵活 |
| **多模型支持** | 字节系模型为主 | 支持主流模型 |
| **发布渠道** | 多（微信、飞书、抖音） | API+网页嵌入 |
| **价格** | 免费额度+付费 | 开源免费+云付费 |
| **中文文档** | 完善 | 完善 |

---

## 推荐选择

### 选择 Coze（扣子）如果：
- ✅ 希望快速部署，国内用户访问
- ✅ 需要发布到微信、飞书等平台
- ✅ 希望使用字节系的模型（如豆包）
- ✅ 需要丰富的插件生态

### 选择 Dify 如果：
- ✅ 需要更多模型选择（GPT-4、Claude等）
- ✅ 需要自部署，数据隐私要求高
- ✅ 需要强大的向量检索功能
- ✅ 需要定制化API服务
